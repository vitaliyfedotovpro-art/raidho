# Доказательство: расход падает с повторением (авто-дистилляция)

Реальный замер на живом DeepSeek (`deepseek-chat`), одна и та же read-only задача
×5, дважды: без авто-дистилляции (бейзлайн) и с ней. Токены — из API-поля `usage`.
Скрипт: `benchmarks/autodistill_curve.py`. Цены сверены 2026-06-11
($0.14/M вход, $0.28/M выход).

## Кривая (токены вход / выход на прогон)

| run | baseline | distill |
|---|---|---|
| 1 | 2254 / 88 | 2402 / 140 ← учит процедуру (+safety-verify) |
| 2 | 2260 / 92 | **225 / 15** ← детерминированный реплей |
| 3 | 2260 / 92 | **225 / 15** |
| 4 | 2260 / 92 | **225 / 15** |
| 5 | 2260 / 92 | **225 / 15** |

## Итог

- **Бейзлайн:** каждый прогон ~$0.00034 (всегда полная LLM-петля).
- **Distill, прогон 1:** $0.00038 — небольшая надбавка за обучение (+ один
  safety-verify вызов). Окупается со 2-го прогона.
- **Distill, установившийся режим (прогоны 2..5):** $0.00004 — **×9.6 дешевле**
  на повтор (вход ×10, выход ×6).
- **Кумулятивно за 5 прогонов: 70% экономии** ($0.00171 → $0.00052).

## Почему так

Полная петля переоплачивает растущий контекст на каждой итерации (≈3 round-trip:
list_dir → read_file → ответ, каждый тащит system+tools+history). Выученная
процедура заменяет петлю на детерминированный сбор (0 LLM) + один синтез-вызов
(225 вход / 15 выход). Чем чаще повторяется задача, тем ближе кумулятивная
экономия к ×9.6.

## Перспектива (как это масштабируется)

Экономия на повтор растёт с «тяжестью» петли:
- мелкая задача (эта, 2 чтения, ~2.3k токенов) → ×~10 на повтор;
- крупный аудит (эксп. 11.06: 8 итераций, 45k токенов петлёй) → реплей убирает
  именно эти итерации, абсолютная экономия в разы больше.

⚠️ Граница: только read-only задачи (записи остаются на LLM-пути — безопасность);
кривая «вниз» начинается со 2-го повтора и применима к повторяющимся задачам
(аудиты, инспекции, «сколько/где X»), не к разовым.
